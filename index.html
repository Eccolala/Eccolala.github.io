java static initialization bloks

Posted on 2016-10-14   |  

static 修饰符 的职责
static 关键字用来创建属于某个类的字段或者方法，而不是用来创建类的实例

为什么使用她？
当一些变量创建时，她们都有类的完整的实例变量的副本，存储在内存中的不同位置，
有时你会想要创建一些变量使用同一个副本，static 就派上用场了

如何调用实例变量
类名.变量名

在实例方法中引用实例变量会有一些限制，下面是一些实例变量，类变量以及方法的组合
实例方法可以直接访问实例变量和别的实例方法
实例方法可以直接访问类变量和类方法
类方法可以直接访问类变量和类方法
实例方法不可以直接访问实例变量和实例方法，类方法中不能使用 this 关键字

static 的其他用途
使用 static final 定义常量

为什么要使用 static initializatioin blocks ?
一般对字段的初始化会有局限性，因为你无法添加逻辑（错误处理，或者填充一个复杂的数组），实例变量可以在构造器中初始化并且添加一些错误处理，静态代码块类似于静态方法，静态方法可以方便重用

如何使用 static initializatioin blocks ?
在一个类中，你可以使用任意数量的静态初始化块，她们可以出现在类的任意位置，runtime system 会以她们出现的顺序调用她们。
